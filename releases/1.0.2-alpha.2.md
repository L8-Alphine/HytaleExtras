# HyExtras 1.0.2-alpha.2

**Target:** Hytale Server `0.5.x` (built on `0.5.6`) · **Channel:** Alpha

A **fine-tuning** release that adds per-entity granularity, opt-in persistence, richer conditions, a
change-event API, and — the headline — **real NPC & mob interaction**. Everything is
**backward-compatible**: every default preserves the previous behavior, so existing maps keep working
after you drop the jar in.

> ⚠️ **Alpha build.** Tested against Hytale `0.5.6`. Please back up your world/config and report issues.

## 🌟 Highlights
- **Interact with NPCs & mobs.** Right-click/use on an entity now fires your interaction triggers and
  hands you the entity to act on — give it a tag, read its variables, hide it, and more.
- **Variables can persist.** Opt in globally, or persist single keys with a `persist:` prefix.
- **Smarter conditions.** New operators: `greater_or_equal`, `less_or_equal`, `divisible_by`,
  `contains`, `regex`.
- **Tunable timing.** The visibility sync, interaction replay window, and TagNPC retention are no
  longer hardcoded.

## ✨ New
- NPC/Mob interaction: the interacted entity is exposed as the `hextras_interacted_entity` variable and
  a new `Target: "interacted_entity"` option on every TagNPC effect/condition.
- Opt-in player-variable persistence (`playerVariablesPersistent`, or the `persist:` key prefix).
- Five new variable operators on `variable_condition` and `tag_npc_variable_condition`.
- Conditional interaction prompts — volume tag `hextras:interaction_prompt_conditional=true` shows the
  prompt only when the effect chain's conditions pass.
- Developer display names for TagNPC entities.
- Eight new `hyextras.properties` keys (see Configuration).

## 🔧 Improved
- Visibility sync interval, interaction replay window, and TagNPC state retention are now configurable
  (and re-applied on `/hextras reload` where applicable).
- Player tag/variable saves are now atomic and run off the main thread, with debounced coalescing;
  disconnect saves still flush immediately so nothing is lost.

## 🐛 Fixed
- TagNPC entity display name field is now functional instead of unused.
- `player_hide_entity` with `TargetSelector=entities` no longer spams the console on normal use.
- Non-integer `PartyAmount` volume tags are handled cleanly instead of being silently ignored.

## 👩‍💻 For developers
- New change-event API: `HyExtrasApi.subscribe(...)` for tag, variable, visibility, and interaction
  changes — no more polling.
- API-capability notes for Hytale `0.5.6`: NPC interaction is available; per-viewer **nameplate**
  override is feasible (planned), while glow/team/equipment/fake-spawn are not exposed by the platform.

## ⚠️ Not yet implemented (planned)
Per-viewer nameplate override, packet batching, NPC-vs-mob classification, TagNPC disk persistence,
cooldown scopes, and tag metadata.
